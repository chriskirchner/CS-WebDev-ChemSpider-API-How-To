Getting Started

Many of ChemSpider's SOAP methods require security tokens while many of their REST methods don't. Just to be safe, grab a token from their Registration Pageby first getting an RSC ID (Royal Society of Chemistry), merging a ChemSpider account, and extracting the security token from the Profile Link they mention on page zero. It may be difficult to get to the Profile page from your actual account, counterintuitively; the link Profile Page link may keep you sane.

Of course, before you start casting spells at the ChemSpider, Cross-Origin Resource Sharing prevents wizards from mixing potions using connections from different internet domains. If you are considering a Subscriber Service for your mighty guild of applications, consider giving the good old guys at the Royal Chemical Society a call to get cross sharing access. For testing purposes, making connections from your local server works fine, granted you have something like XMLHttpRequest installed server-side when using javascript. While XMLHttpRequest usually comes with browsers so that it's connections can be rejected by its own CORS anti-browsing function, XMLHttpRequest usually doesn't come with server's where it can run freely. It's quite ironic really, or i've had one too many potions today.

For those inclined for Node.JS, it behooves you to install XMLHttpRequest through NPM (Node Package Module):

npm install xmlhttprequest

Then later, after you forgot what you just installed, you can imbed XMLHttpRequest in your infamous script:

var XMLHttpRequest = require('xmlhttprequest').XMLHttpRequest;
var request = new XMLHttpRequest();
